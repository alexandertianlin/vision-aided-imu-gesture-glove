# Work Log: V2.2 Full-Chain Debug — 2026-06-22

## Problem

V2.2 Python 端成功检测到手势并输出关节角度（`ANGLE seq=... p=... y=...`），
但 Unity 3D 手没有响应。

## Root Cause

信号链中两个依赖 IMU 的阻塞点：

### 阻塞点 1: `HandMotionManager.IsCalibrated`
```
MediaPipeAngleReceiver.LateUpdate():
  if (!handMotion.IsCalibrated) → return;  // 直接跳过
```
IMU 手套未连接 → `SerialReceiver.ImuDataDict` 为空
→ 按 Space 也无法校准 → `IsCalibrated` 一直为 false
→ MediaPipeAngleReceiver 什么都不做

### 阻塞点 2: `FingerSolver.MoveVisionAngleAnchorAndSyncImu()`
```
MoveVisionAngleAnchorAndSyncImu(pitch, yaw, blend, imuQuat, ...)
                                     需要 IMU 四元数 ↑
```
没有 IMU 数据 → `relativeImuRot` 为空/无效
→ `ExtractFingerAngles()` 无法提取角度
→ 视觉角度不会被应用

## Fix Applied

给 `MediaPipeAngleReceiver.cs` 新增 `visionOnlyMode` 模式：

1. **新增开关** `public bool visionOnlyMode = false`
2. **跳过 IsCalibrated** 当 visionOnlyMode = true 时，不要求 IMU 校准
3. **替换 API 调用** visionOnlyMode 下使用 `ForceVisionAngleAnchor()`
   （不需要 IMU 四元数，直接设定 currentPitch/currentYaw 并应用姿态）
4. **跳过超时检查** vision-only 模式下不检查 UDP 超时（没有 IMU 接管）

## V2.2 使用方法

MediaPipeAngleReceiver 现在会自动检测 IMU 是否存在：
- **有 IMU 手套** → 自动融合模式（需要按 Space 校准）
- **无 IMU 手套** → 自动回退到纯视觉模式（无需校准，即开即用）
- 强制纯视觉：Inspector 中 `visionOnlyMode = true`

Python 端统一命令：
```cmd
python mediapipe_udp_sender_v2_2.py --skip-cal
```

无需手动切换，插上手套就融合，拔掉就纯视觉。

## Calibration Database

`finger_calibration_db.json` 已保存完整数据（5指各100样本），
可通过 `--skip-cal` 参数跳过每次启动的100秒标定。

## Current Signal Chain (Fixed)

```
Python V2.2
  → D435i → MediaPipe Hand → compute_finger_angles()
  → UDP port 5058
  → Unity MediaPipeAngleReceiver
  → visionOnlyMode? 
      YES → ForceVisionAngleAnchor() ← No IMU needed
      NO  → MoveVisionAngleAnchorAndSyncImu() ← IMU fusion
  → FingerSolver.ApplyFingerPose()
  → 3D hand moves ✓
```
