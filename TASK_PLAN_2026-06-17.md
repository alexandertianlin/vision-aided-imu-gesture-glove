 ### 下午 13:30 — 16:30：六个模型并行 Benchmark

 #### GPU 动态调度方案

 ```text
 第一批（同时启动）：
 GPU A — RTMPose-Hand（跑通概率 ★★★★★）
 GPU B — AWR-Net（项目需求 ★★★★★）
 GPU C — HandPointNet（环境风险最高，先踩坑）

 动态调度（谁先跑完谁接新任务）：
 第一个空闲 → YOLO-Hand-Pose（跑通概率 ★★★★★）
 第二个空闲 → IPNet（项目需求 ★★★★）
 第三个空闲 → MMPose-Hand（跑通概率 ★★★★）
 ```

 #### 淘汰机制
 每个模型部署开通：
 - ⏱ 30 分钟部署计时
 - 超时 → 记录为 "Fail - Environment incompatible" 并跳过
 - 例如 HandPointNet（Python 2 / PyTorch 0.3 / CUDA 8）如果半小时内跑不通就直接放弃

 #### 每个模型必须记录
 | 指标 | 说明 |
 |------|------|
 | Detect Rate | 模型正确检测到手套手的帧比例 |
 | FPS | 推理帧率，排除 I/O 时间 |
 | Latency | 单帧推理延迟（ms） |
 | Stability | 连续帧的关键点抖动程度（1-5） |
 | 21 Keypoints | 是否输出 21 个标准手部关键点 |
 | 3D Support | 是否输出 3D 坐标 |
 | Deployment Time | 从开始部署到成功运行的时间 |
 | Failure Reason | 如果失败，记录具体原因 |
